## 1. 快速压缩上下文、节约 Token

**核心思想**：用一份"项目记忆文件"替代长对话回滚，让 Cascade 每次只读一个短文件即可恢复状态。

### 1.1 标准开场白（推荐复制）
```
继续 f:\AIsaibotumu\Test-1 钢筋 3D 项目，先读 PROGRESS.md
```
- 让 AI **只读一个文件**就能恢复上下文，比翻历史对话省 90% token。
- 关键：`PROGRESS.md` 必须保持精简（< 200 行），结构化分节（目标 / 已完成 / 待办 / 约定）。

### 1.2 项目记忆文件维护守则
- **一个项目一份** `PROGRESS.md`（或 `progress.txt`），放项目根目录。
- 写**结论**不写过程：保留"已完成清单"+"待办优先级"，不保留调试细节。
- 每完成一个里程碑就让 AI 帮你更新它：
  ```
  把今天做的 XXX 加到 PROGRESS.md 的"已完成"，把对应 todo 划掉
  ```
- 文件结构变化时同步更新"文件结构"小节，避免 AI 用 `list_dir` 反复探查。

### 1.3 其他省 token 技巧
- **明确文件路径**：直接给 `f:\...\BeamScene.tsx` 比让 AI 搜索省很多。
- **指定行号**：`改 BeamScene.tsx 第 120-140 行的标注` → AI 不用读全文件。
- **关掉不相关的打开标签**：IDE 元数据会带进上下文。
- **新对话起步**：长对话超过 30 轮后开新会话 + 让 AI 先读 `PROGRESS.md`。
- **不要让 AI 回显大段代码**：要它"直接改"而不是"先给我看改成什么样"。

## 2. 快速打开本地网页 http://127.0.0.1:12306
### 1. 启动服务
cd f:\AIsaibotumu\Test-1
npm run dev
### 2. 打开 IDE 内置预览
{
  "Url": "http://localhost:5173",
  "Name": "钢筋 3D 可视化"
}
### 2.1 让 Cascade 唤起 IDE 内置预览
直接说：
```
帮我打开 http://127.0.0.1:12306 的浏览器预览
```
Cascade 会调用 `browser_preview` 工具，在 IDE 内嵌一个预览面板，**控制台日志可以直接回流给 AI 调试**（比外部浏览器更有用）。

### 2.2 手动方式
- **PowerShell**：`Start-Process "http://127.0.0.1:12306"`
- **Win+R** → 粘贴 `http://127.0.0.1:12306` → 回车
- **VSCode 命令**：`Ctrl+Shift+P` → `Simple Browser: Show`

### 2.3 本项目 dev 服务器
```powershell
npm run dev    # 默认 http://localhost:5173
```
端口冲突时 Vite 会自动 +1（5174、5175...），看终端输出。

## 3. 快速更新 GitHub 上的代码

### 3.1 一行命令推送（最常用）
```powershell
git add -A; git commit -m "更新说明"; git push
```
推送后 GitHub Actions 自动构建并部署到 `https://dj78b5zmzp-tech.github.io/Test-01-3D/`（约 1-2 分钟）。

### 3.2 让 Cascade 代办
```
把当前改动提交并推送，commit message: "xxx"
```

### 3.3 代理相关（本机用 Clash Verge 7897 端口）
已配置过一次，全局生效，无需重复：
```powershell
git config --global http.https://github.com.proxy http://127.0.0.1:7897
```
**临时关闭代理**（公司网/直连时）：
```powershell
git config --global --unset http.https://github.com.proxy
```

### 3.4 查看部署状态
- Actions 日志：https://github.com/dj78b5zmzp-tech/Test-01-3D/actions
- 部署成功后浏览器**强制刷新** `Ctrl+F5` 清缓存。

### 3.5 拉远端最新代码
```powershell
git pull --rebase
```
（`--rebase` 让本地 commit 排在远端之后，历史更干净）

## 4. Cascade 常用模式 / 提示词模板

### 4.1 模式选择
- **Ask（只读）**：分析、问问题、看代码，不动文件。
- **Code（写）**：直接改文件、跑命令。
- 切换：消息框下方下拉，或快捷键。

### 4.2 高效指令模板
| 场景 | 指令 |
|------|------|
| 恢复上下文 | `继续 xxx 项目，先读 PROGRESS.md` |
| 最小改动修 bug | `用最小改动修复 XXX，不要重构其他地方` |
| 加功能 | `参考 useBeamRebars.ts 的模式，给柱加锚固` |
| 类型检查 | `跑 npx tsc -b --pretty false 看看有没有报错` |
| 更新进度文件 | `把刚才做的加进 PROGRESS.md，对应 todo 划掉` |
| 提交推送 | `提交并推送，commit: "xxx"` |

### 4.3 让 AI 少废话
- 用户全局规则已设置中文 + 简洁直接，无需每次重复。
- 不想要解释只想要结果：`直接改，不要解释`。

## 6. Windsurf / Cascade 常用指令

### 6.1 全局快捷键（Windows）
| 快捷键 | 作用 |
|--------|------|
| `Ctrl + L` | 打开 / 聚焦 Cascade 对话面板 |
| `Ctrl + I` | 行内 Command（在编辑器里直接对选中代码下指令） |
| `Ctrl + Shift + L` | 新开一个 Cascade 会话（清空上下文，省 token 利器） |
| `Ctrl + Enter` | 发送消息 / 接受建议 |
| `Esc` | 中断 Cascade 当前生成 |
| `Tab` | 接受 Supercomplete / 自动补全 |
| `Ctrl + →` | 逐词接受补全 |
| `Alt + ]` / `Alt + [` | 切换补全候选 |
| `Ctrl + Shift + P` | VSCode 命令面板（Windsurf 基于 VSCode） |

### 6.2 @ 提及（@-mention）—— 精确投喂上下文
在对话框输入 `@` 触发：

| 提及 | 作用 |
|------|------|
| `@文件名` | 把指定文件丢进上下文（比让 AI 自己搜更省 token） |
| `@文件夹` | 整个目录作为上下文 |
| `@function/类名` | 仅引用某个符号 |
| `@web` | 触发联网搜索 |
| `@docs` | 查官方文档（React、three.js 等） |
| `@terminal` | 把上一条终端输出拉进来 |
| `@past-conversation` | 引用历史会话片段 |

**省 token 要诀**：宁可 `@BeamScene.tsx` 也别说"那个梁场景文件"，AI 不用再 grep。

### 6.3 斜杠命令 / 工作流
- 输入 `/` 触发；项目里 `.windsurf/workflows/*.md` 定义的工作流会出现在列表。
- 想固化重复操作（如"打包+提交+推送"）可以让我帮你写一个 workflow 文件，下次直接 `/部署` 调用。

### 6.4 模式（Mode）
- **Write**：可改文件、跑命令（默认）。
- **Chat / Ask（只读）**：分析、答疑，不改文件——讨论方案时用，避免误改。
- **Legacy**：旧版交互。
- 切换：对话框左下方下拉。

### 6.5 Memories（记忆系统）
- 让我**长期记住**某事：
  ```
  记住：本项目所有距离单位用 mm
  ```
- 让我**忘记**：
  ```
  忘掉关于 XXX 的记忆
  ```
- 全局规则已生效：中文交流 + 不泄露个人信息。
- 项目级规则可写在 `.windsurfrules` 文件里（自动注入）。

### 6.6 上下文 / Token 管理
- **顶部 token 进度条**：长对话超过 ~70% 时建议 `Ctrl+Shift+L` 开新会话 + 让我先读 `PROGRESS.md`。
- **关闭无关编辑器标签**：打开的文件会进入元数据。
- **折叠 Cascade 历史**：点会话标题可隐藏旧消息（仅 UI 折叠，不真删）。

### 6.7 工具调用相关
- 我可以并行调用多个独立工具（读文件、搜索），不用你催。
- 命令行有"自动批准白名单"，在设置 → Cascade → Allowlist 里加常用安全命令（如 `git status`、`npm run dev`）即可免确认。
- **永远不要**把 `rm -rf`、`git push --force` 加白名单。
